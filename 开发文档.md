# 番茄钟软件开发文档

## 1. 文档目标

本文档用于指导番茄钟软件 MVP 的开发实现。当前信息已经足够支撑第一版开发，未确定项仅影响交互细节，不影响整体架构、模块拆分和实现顺序。

技术选型已确定为 `PySide6`，目标平台为 Windows，后续预留 Linux 兼容空间。

## 2. 开发目标

第一阶段只完成以下能力：

1. 圆形悬浮球常驻显示，支持拖动、置顶、显示倒计时
2. 专注 25 分钟、短休息 4 分钟、每 4 轮后长休息 8 分钟
3. 专注结束后弹出多显示器全屏休息提醒
4. 支持跳过提醒、延后 5 分钟、关闭提醒、直接开始下一轮
5. 点击悬浮球打开主界面
6. 主界面支持开始、暂停、重置、设置
7. 关闭主界面后最小化到托盘
8. 支持开机自启与本地配置持久化

## 4. 核心架构

建议采用“应用协调层 + 计时内核 + UI 层 + 系统适配层”结构。

### 4.1 应用协调层

由 `app.py` 负责：

1. 初始化配置
2. 创建计时器引擎
3. 创建悬浮球、主窗口、全屏提醒窗口、托盘
4. 连接信号与槽
5. 控制不同窗口之间的状态同步

### 4.2 计时内核

由 `core/` 负责：

1. 维护专注 / 短休息 / 长休息 / 暂停状态
2. 提供剩余时间计算
3. 负责周期切换
4. 负责延后 5 分钟逻辑
5. 不直接依赖具体 UI 组件

### 4.3 UI 层

由 `ui/` 负责：

1. `floating_ball.py` 负责圆形悬浮球
2. `main_window.py` 负责主控制界面
3. `fullscreen_break.py` 负责全屏休息提醒
4. `styles.py` 统一颜色、圆角、阴影、字体、控件风格

### 4.4 系统适配层

由 `system/` 负责：

1. 托盘能力
2. 开机自启
3. 多显示器信息获取
4. 配置读写

## 5. 计时状态机设计

建议使用显式状态机，不要把状态切换散落在窗口代码中。

### 5.1 状态定义

1. `IDLE`：未开始
2. `FOCUS`：专注中
3. `SHORT_BREAK`：短休息中
4. `LONG_BREAK`：长休息中
5. `PAUSED`：暂停中

### 5.2 关键规则

1. 默认进入 `FOCUS` 时长 25 分钟
2. 每完成 1 次专注，计数加 1
3. 完成 4 次专注后，下次进入 `LONG_BREAK`
4. 其他情况进入 `SHORT_BREAK`
5. 休息结束后回到 `IDLE`，由用户手动开始下一轮
6. 延后 5 分钟本质上是对当前提醒增加一个新的等待周期

### 5.3 计时实现原则

不要使用阻塞式 `sleep` 驱动主逻辑。推荐做法：

1. 使用 `QTimer` 每秒刷新显示
2. 使用“目标结束时间戳 - 当前时间”计算剩余时间
3. 即使窗口卡顿或系统挂起，恢复后也能校正剩余时间

## 6. 窗口设计

### 6.1 悬浮球

要求：

1. 无边框
2. 圆形
3. 始终置顶
4. 可拖动
5. 显示剩余时间
6. 点击打开主窗口

建议：

1. 尺寸不要过小，优先保证点击和拖动命中
2. 默认半透明，悬停时增强可见性
3. 专注和休息状态用不同颜色区分

### 6.2 主窗口

要求：

1. 居中显示
2. 大约占屏幕四分之一
3. 展示当前状态、剩余时间、番茄轮次和设置入口

建议：

1. 无需真正全屏
2. 采用圆角、大留白、柔和阴影
3. 关闭时不退出，只隐藏到托盘

### 6.3 全屏休息提醒

要求：

1. 覆盖所有显示器
2. 尝试抢焦点
3. 若抢焦点失败，仍显示清晰提示文案
4. 提供跳过、延后 5 分钟、关闭提醒、开始下一轮

建议：

1. 每个屏幕创建一个无边框全屏窗口
2. 主交互按钮保持少量且大尺寸
3. 倒计时视觉层级最高

## 7. 配置与持久化

建议使用 JSON 进行本地配置持久化，首版即可满足需求。

建议配置项：

```json
{
  "focus_minutes": 25,
  "short_break_minutes": 4,
  "long_break_minutes": 8,
  "cycles_before_long_break": 4,
  "floating_always_on_top": true,
  "floating_position": {"x": 120, "y": 120},
  "launch_on_startup": true,
  "allow_multiple_snooze": true
}
```

Windows 下建议存储在用户目录的应用配置位置，不要写死到仓库目录。

## 8. 开发顺序

推荐按以下顺序推进：

1. 搭项目骨架、依赖和入口
2. 完成计时状态机与配置模块
3. 完成悬浮球窗口
4. 完成主窗口
5. 完成全屏提醒窗口
6. 接入托盘与开机自启
7. 做多显示器、焦点、最小化场景测试
8. 最后统一视觉样式与细节打磨

## 9. 测试重点

优先覆盖以下内容：

1. 状态切换是否正确
2. 长时间运行后计时是否漂移
3. 多次延后是否符合预期
4. 主窗口关闭后是否仍驻留托盘
5. 多显示器下全屏提醒是否全部覆盖
6. 应用重启后配置和悬浮球位置是否恢复

## 需要优化的地方

当前的 UI 细节不够精致，使用了太多系统默认样式的控件。我需要你帮我修改 UI 相关的代码（包括布局、样式表 QSS 等），将其升级为更具现代感、类似 Mac OS 风格的精美界面。

具体的 UI 优化需求如下，请逐一落实到代码修改中：

字体与排版层级优化：

当前字体显得过小且单薄。请调整全局字体设置，使用更具现代感的无衬线字体（如系统默认的 UI 字体、PingFang SC 或 Microsoft YaHei）。

加粗核心信息：“待开始”及“01:00”倒计时需要更粗的字重（如 font-weight: 600/bold），增加不同文本之间的字号差异，形成明确的视觉层级。

重构时长输入框（核心痛点）：

现有的数值输入框（SpinBox）太大，自带的上下箭头非常突兀且不美观。

修改方案：隐藏默认的上下箭头。将输入框的高度调小，使其与左侧的文字（如“专注时长”）垂直居中对齐。

样式上，给输入框加上细致的浅色边框和轻微的圆角，背景色可设置为纯白或微透明，使其看起来像现代 Web 或 Mac OS 的输入组件。如果可以，请在输入框两侧或内部使用自定义的精美小巧的 + - 符号替代传统箭头。

添加作者署名：

在应用界面的最底部（居中或右下角），添加一行小字：“by sikm”。

样式要求：字体要小，颜色使用低透明度的灰色或浅棕色（不要喧宾夺主），起到点缀作用即可。

Mac OS 风格化与留白（Padding & Margin）：

全局圆角：为所有的按钮（开始、暂停、重置）和输入框增加统一的圆角大小（border-radius，例如 6px 到 8px），让视觉更柔和。

间距调整：下方的设置区域（时长设置和复选框）当前排版有些松散且对齐不够完美。请统一各行元素的高度，增加行与行之间、以及左右两侧的边缘留白（Padding），使其像一个整齐的设置面板。

按钮交互：为“开始”、“暂停”、“重置”按钮增加鼠标悬停（Hover）和按下（Pressed）时的颜色微调反馈，提升交互质感。
关于对齐：你可以看到图中“专注时长”、“短休息”、“长休息”这三个词的字数不一样。建议在代码里给它们设定一个固定的宽度（比如让它们右对齐），这样后面的输入框就会在一条完美的垂直线上，强迫症看着会非常舒服。

关于复选框：系统默认的方形复选框（Checkbox）也有点生硬。在现代UI中，通常会把这种布尔值的设置做成 Mac OS 那种椭圆形的开关（Switch Toggle），AI 助手也能很容易地用代码帮你把 Checkbox 渲染成 Toggle 开关的样子，你可以让它试试看！
